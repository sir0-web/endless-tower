# ユーザー状態の閲覧（ADMIN）セットアップ手順

ADMINの「ユーザー管理」で名前検索すると、そのプレイヤーの**現在のステータス・装備**を閲覧できる機能です。
プレイ中の心拍（すかるぽりん heartbeat、約30秒ごと）に状態スナップショットを相乗りさせ、
既存テーブル `active_sessions` の新カラム `state`(jsonb) に保存します。
ADMINは Service Role Key 経由のAPI（`/api/admin-event` の `player_state`）で読み出します。

## 1. `active_sessions` に `state` カラムを追加（デプロイ前に実行）

Supabase → SQL Editor で実行：

```sql
alter table public.active_sessions
  add column if not exists state jsonb;
```

> **重要**：コードをデプロイする前にこのSQLを流してください。
> なお、未追加でも安全側に倒しています——状態保存は心拍本体（スポーン/コマンド/報酬）とは
> 別更新＋エラー握りつぶしのため、カラムが無くても心拍は壊れません（状態が保存されないだけ）。
> その間ADMINの状態表示は「同期データなし」になります。

## 2. 環境変数

追加不要。既存の `VITE_SUPABASE_*` / `SUPABASE_SERVICE_ROLE_KEY` / `VITE_ADMIN_KEY` を流用します。

## 3. 使い方

1. `/admin` → 「ユーザー管理」タブ → プレイヤー名で検索（部分一致）
2. 最上部に「現在のステータス・装備」カードが表示されます
   - Lv / EXP / 到達F / HP / スタミナ / ターン / 未割振Pt
   - STR/AGI/DEX/INT/VIT/LUK
   - 装備（スロット別）・魔法の書・所持品（回復/コイン）・バッグ内装備
   - 「最終同期」時刻つき

## 仕様・注意

- 表示されるのは**最終同期時点のスナップショット**です。プレイ中はほぼリアルタイム（約30秒間隔）、
  オフラインのときは最後に同期された内容になります。
- セーブ自体は引き続き各端末の localStorage に保存されます（この機能はあくまで閲覧用の同期です）。
- 同名の複数セッションがヒットした場合は、最終同期が新しい順に最大20件まで表示します。
- イベントフロア／ゲームオーバー中は心拍を送らないため同期されません（既存仕様）。
